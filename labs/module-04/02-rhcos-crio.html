Immutable updates: a MachineConfig rollout

A Mobily admin needs a new kernel argument on every worker. On RHCOS you don't ssh + vi — you apply a MachineConfig and the Machine Config Operator (MCO) rolls it out node-by-node: cordon → drain → reboot into the new image. Watch capacity stay safe.

0/3 updated never more than 1 node down
The MCO updates one node at a time, draining its pods first, so the subscriber-api keeps serving throughout. Each node reboots into a brand-new, identical OS image — no drift, fully reproducible.

Why CRI-O, not Docker?

CRI-O does exactly one job: be the runtime Kubernetes drives. Less surface area, fewer CVEs, versioned with the cluster.
