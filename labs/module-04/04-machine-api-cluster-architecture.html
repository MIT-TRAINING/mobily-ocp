Concept · Declarative infrastructure

Nodes are cattle too. The Machine API herds them.

The same reconcile loop that manages pods manages the cloud VMs that become your nodes. A MachineSet is the ReplicaSet for Machines. Scale it, meet the node roles, and let the autoscaler grow the fleet for pending pods.

MachineSet → Machines → Nodes

Mobily needs more CDR-processing capacity. Scale the cdr-workers MachineSet — the Machine API provisions VMs, boots RHCOS on each (via Ignition), and joins them as worker Nodes. No clicking in the cloud console.

Machine = one host that should exist; MachineSet = a template + a count (the ReplicaSet for Machines). Delete a Machine and the set provisions a replacement — that's worker-node self-healing.

Autoscaling: pending pods grow the fleet

An evening traffic spike creates pods that can't be scheduled (no room). The ClusterAutoscaler notices and raises the MachineSet; new nodes appear; the scheduler places the pods.

idle · 2 nodes
Machine/MachineSet are to nodes what Pod/ReplicaSet are to containers — the reconcile loop, one layer down. A MachineAutoscaler sets min/max; the ClusterAutoscaler scales to fit pending pods, then shrinks when demand falls.
