Four hands-on visualizations that make Module 4 click — how OpenShift extends Kubernetes into a finished product (stack, portfolio, history), what the immutable node really runs (RHCOS & CRI-O), how the platform runs itself with the Operator pattern (Cluster Operators + the Cluster Version Operator), and how the Machine API manages the node fleet declaratively. Each ends with a short self-check quiz.

How to use this

Work through the four in order (each builds on the last) or jump to any concept. Everything runs in your browser — no install, no internet, no cluster needed. What you'll be able to explain afterwards

  • Exactly what OpenShift adds on top of Kubernetes — and why each layer (OS, runtime, ingress, auth, registry, monitoring, console) exists
  • The v3 → v4 self-managing shift, and how to place every member of the product portfolio (OCP, ROSA/ARO, SNO, MicroShift, Sandbox, OKD)
  • Why nodes run immutable RHCOS (you declare a MachineConfig; the MCO rolls it out) and why the runtime is CRI-O, not Docker
  • The Operator pattern (CRD + CR + reconcile loop) and how OpenShift runs itself via Cluster Operators coordinated by the CVO
  • How the Machine API applies the reconcile loop to infrastructure, and the control / worker / infra node roles
