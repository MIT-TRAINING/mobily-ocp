# Exercise 2 — Routes & TLS Termination

> **Scenario:** The **self-care portal** needs to be reachable by subscribers on the
> internet — first over plain HTTP to prove the path, then over **HTTPS with edge TLS**.
> You'll also reason about when you'd choose **passthrough** or **re-encrypt**, and see
> that a Kubernetes **Ingress** becomes a Route.

## Tasks

1. Deploy **`self-care`** (`ubi9/httpd-24`) and give it a ClusterIP Service on 8080.
2. Create an **unsecured (HTTP) Route** with an auto hostname, and `curl` it (expect
   **403** — connectivity proven).
3. Replace it with an **edge** Route that **redirects HTTP→HTTPS**; confirm `TERMINATION=edge`
   and `curl -k` the HTTPS URL.
4. In one command, list **all Routes** with their **TLS termination** column.
5. Create a Kubernetes **Ingress** for the same Service and show that a **Route is
   generated** to fulfil it.
6. Write one sentence each: when you'd use **passthrough** vs **re-encrypt** instead of
   edge.

> **Hint (Task 3):** `oc create route edge self-care --service=self-care --insecure-policy=Redirect`.

> **Hint (Task 4):** `oc get routes -o custom-columns=NAME:.metadata.name,HOST:.spec.host,TLS:.spec.tls.termination`.

## Reflection

- In **edge**, is the router→pod hop encrypted? Why is that acceptable for an internal
  network but not for PCI-scoped data?
- Which TLS mode is the *only* one that supports **mutual TLS**, and what do you give up
  by choosing it?
- On OpenShift, what actually serves a Kubernetes **Ingress**?

<details>
<summary><strong>✅ Solution</strong> (try the tasks first)</summary>

```bash
# 1. Deploy + Service  (use `oc create deployment`, NOT `oc new-app` — new-app
#    auto-creates a Service, which makes the next `oc expose deployment` fail)
oc create deployment self-care --image=registry.access.redhat.com/ubi9/httpd-24:latest
oc rollout status deploy/self-care
oc expose deployment self-care --port=8080

# 2. HTTP Route + curl
oc expose service self-care
curl -s -o /dev/null -w "%{http_code}\n" http://$(oc get route self-care -o jsonpath='{.spec.host}')/   # 403

# 3. Edge Route (HTTP redirects to HTTPS) — wait for the router to admit it
oc delete route self-care
oc create route edge self-care --service=self-care --port=8080 --insecure-policy=Redirect
sleep 15                                                                                 # router reconcile (else transient 503)
oc get route self-care -o jsonpath='TERMINATION={.spec.tls.termination}{"\n"}'           # edge
EHOST=$(oc get route self-care -o jsonpath='{.spec.host}')
curl -sk -o /dev/null -w "https: %{http_code}\n" https://$EHOST/                         # 403 (reached pod over TLS)
curl -s  -o /dev/null -w "http:  %{http_code}\n" http://$EHOST/                          # 302 (redirect to https)

# 4. All routes + TLS column
oc get routes -o custom-columns=NAME:.metadata.name,HOST:.spec.host,TLS:.spec.tls.termination

# 5. Ingress becomes a Route  (derive the apps domain from an existing route's HOST —
#    a normal user cannot read ingresses.config/cluster)
DOMAIN=$(oc get route self-care -o jsonpath='{.spec.host}' | cut -d. -f2-)
oc create ingress selfcare-ing --rule="selfcare.$DOMAIN/*=self-care:8080"
oc get route          # a Route generated by the Ingress Operator appears

# 6. passthrough  -> pod terminates TLS (mutual TLS / cert pinning); no L7 path routing.
#    re-encrypt    -> encrypted end-to-end AND router can route by host/path (PCI + routing).

# Cleanup
oc delete route --all ; oc delete ingress selfcare-ing 2>/dev/null ; oc delete all -l app=self-care
```

**Verified output** *(learner25 · OCP 4.18 · project `l25-m6d1`):*

```
403
TERMINATION=edge
https: 403
http:  302
NAME              HOST                                                                 TLS
self-care         self-care-l25-m6d1.apps.mobily-ocp-training.ocp.supercloudlabs.com   edge
selfcare-ing-xxx  selfcare.apps.mobily-ocp-training.ocp.supercloudlabs.com             <none>
```

**Key point:** `oc expose service` → HTTP Route; `oc create route edge` → HTTPS at the
router (cleartext to the pod). **passthrough** = pod owns TLS (only mode for mTLS, no L7
routing); **re-encrypt** = encrypted end-to-end *and* routable. A Kubernetes **Ingress**
is fulfilled by an auto-generated **Route** on the same HAProxy router.
