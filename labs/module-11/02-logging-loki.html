Concept · Logging & LogQL

Select the streams, then filter the lines.

Loki stores every container's logs indexed by labels. A LogQL query narrows in two moves: a stream selector {namespace=…} picks which streams, then line filters |= != |~ grep within them. Build a query and watch the log volume collapse from noise to the one line you need.

Build a LogQL query

40 of 40 lines
Start broad, then narrow. Pick a namespace to select streams (Loki uses the label index — fast), then add line filters to grep the text. Filter before you read: you search kilobytes, not terabytes.

Vector (collector, per-node DaemonSet) → LokiStack (store, indexed by labels) → console / LogQL (visualize). A ClusterLogForwarder routes each type to its outputs. oc logs is the live, unstored fallback — great for "right now," useless for history.
