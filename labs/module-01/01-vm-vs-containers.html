Concept · Virtual Machines vs Containers

Same hardware. Two ways to pack it.

A VM ships a full guest OS for every workload. A container shares the host kernel and ships only the app. Deploy workloads to each side and watch what fits.

Pack the host

Each side runs on an identical host with 16 GB RAM. Click “Deploy a workload” and see how many you can fit before you run out.

🖥️ Virtual Machine model

RAM used0 / 16 GB

Each VM = ~2.2 GB (2 GB guest OS + app) · boots in ~40 s · image ~2 GB

📦 Container model

RAM used0 / 16 GB

Each container = ~0.25 GB (shared kernel + app) · starts in ~0.4 s · image ~120 MB
