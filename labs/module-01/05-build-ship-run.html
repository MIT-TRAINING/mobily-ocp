Tags vs digests — naming the artifact

registry.example.com/team/myapp:1.4.2
└──── registry ────┘ └team┘ └app┘ └tag┘
A tag (:1.4.2, :latest) is human-friendly but mutable — latest can point to different content over time.
myapp@sha256:9f86d081884c7d65...
A digest is a content hash — immutable. It always refers to the exact same bytes. Pin production deployments to a version tag or digest, never bare latest.
