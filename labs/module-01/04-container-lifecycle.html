Concept · The container lifecycle

Drive a container through its life

A container moves through a small set of states. Only certain actions are valid from each state — the runtime enforces that. Issue commands and watch it transition (invalid actions are disabled).

Lifecycle state machine

Runningprocess executing
Pausedfrozen in memory

The transitions

createimage → Created

pause / unpauseRunning ⇄ Paused
stop / killRunning → Stopped
rmCreated/Stopped → Removed
startStopped → Running again
A Stopped container still keeps its writable layer (you can restart it). Only rm destroys it for good.
