Installing & administering OpenShift — visualized

Four hands-on visualizations that make Module 5 click — how a cluster gets installed and what topology results, how you verify cluster health in three commands, how the Machine Config Operator changes immutable nodes safely via MachineConfigPools, and how you run the cluster lifecycle (channels & upgrades) and manage Projects, users & RBAC. Each ends with a short self-check quiz.

How to use this

Work through the four in order (each builds on the last) or jump to any concept. Everything runs in your browser — no install, no internet, no cluster needed. Companion reading: guides/module-05-installation-cluster-administration.md. Then practice for real with the guided demos and exercises using oc on the shared OpenShift 4.18 cluster (admin steps are instructor-led).

What you'll be able to explain afterwards

  • The installation models (IPI/UPI/Assisted/Agent/Hosted/managed) and when to use each, and how topology (control/worker/infra) is sized
  • The admin's three-command health check and how to triage a DEGRADED operator or a NotReady node
  • How the Machine Config Operator rolls a MachineConfig across a MachineConfigPool safely, and why you'd pause a pool
  • How the CVO runs the lifecycle — channels (stable/fast/eus) and a supervised, drained upgrade
  • How to manage tenants: Projects + quotas/limits, users/groups, and least-privilege RBAC with oc adm policy
