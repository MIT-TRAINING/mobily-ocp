Concept · Changing immutable nodes

RHCOS is immutable — so you change it with a MachineConfig.

Mobily needs a corporate CA on every worker. You don't SSH in — you apply a MachineConfig to the worker MachineConfigPool, and the MCO rolls it out node-by-node, cordon → drain → reboot, keeping the app up.

Roll a MachineConfig across the worker pool

Set how many nodes may update at once (maxUnavailable), optionally pause the pool, then apply. Watch capacity stay safe.

The pool is UPDATED — every worker runs the current rendered config. Apply a MachineConfig to roll a new one out.

Why pause a pool?

Batch changes: pause, apply several MachineConfigs, then unpause once — the MCO renders them together and reboots each node once, not per change.
Protect a window: pause the worker pool so no reboots happen during a sensitive Mobily billing run; unpause after. Config accumulates safely until then.
oc patch mcp worker --type merge -p '{"spec":{"paused":true}}' · check with oc get mcp (master/worker: UPDATED / UPDATING / DEGRADED).
