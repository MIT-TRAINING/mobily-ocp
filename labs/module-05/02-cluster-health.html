Concept · Day-2 health

"Is the cluster healthy?" — answer it in three commands.

Before every release and upgrade, an admin runs the same check: oc get clusterversion, oc get clusteroperators, oc get nodes. Break something below and watch the dashboard — and your go/no-go verdict — change.

The health dashboard

Inject a fault to see how each command surfaces it. Green on all three = safe to proceed.

✅ GO — all three checks green

$ oc get clusterversion

$ oc get clusteroperators

$ oc get nodes

All green: clusterversion Available & not Progressing, no operator Degraded, every node Ready. Safe to ship or upgrade.

When it's red — where to look next

Operator DEGRADED → oc describe co <name> and read the Conditions message. Also oc get events -A --sort-by=.lastTimestamp.
Node NotReady → oc describe node <n> (kubelet/conditions), oc adm top nodes, oc adm node-logs <n>. Collect oc adm must-gather for support.
