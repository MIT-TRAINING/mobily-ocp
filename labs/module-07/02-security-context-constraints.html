Concept · Secure by default

OpenShift won't run your container as root.

A Security Context Constraint is the admission gate that decides what a pod may run as. The default restricted-v2 refuses root and assigns a random UID. Choose what the pod needs and which SCC its ServiceAccount has — and watch it get admitted or denied.

The SCC ladder (least → most privileged)

restricted-v2non-root · random UID · drops caps — the default
nonroot-v2any non-root UID the image requests
hostmount-anyuidmount host paths + any UID
anyuidrun as any UID incl. root (0)
privilegedeverything — host, root, all caps (node agents only)
Grant, don't edit: you loosen a pod's limits by granting its ServiceAccount a broader SCC (oc adm policy add-scc-to-user anyuid -z my-sa), not by changing the pod. Better still: fix the image to be rootless (UBI/sclorg).
