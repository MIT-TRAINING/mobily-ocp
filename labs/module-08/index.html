Authentication, RBAC & identity — visualized

Three hands-on visualizations that make Module 8 click — how RBAC combines Roles and bindings into least-privilege access (and how to prove it with can-i), how the OAuth server authenticates users through identity providers like htpasswd, and how LDAP / Active Directory / OIDC and group sync wire your corporate directory into cluster access. Each ends with a short self-check quiz.

How to use this

Work through the three in order (each builds on the last) or jump to any concept. Everything runs in your browser — no install, no internet, no cluster needed. Companion reading: guides/module-08-authentication-rbac-identity.md. Then practice for real with the guided demos and exercises using oc on the shared OpenShift 4.18 cluster (IdP config is admin-led).

What you'll be able to explain afterwards

  • The four RBAC objects (Role/ClusterRole × RoleBinding/ClusterRoleBinding) and when to use each
  • How a rule is built from apiGroups × resources × verbs, and how to author a custom Role
  • How to design least-privilege access — default roles, bind to groups, project scope, verify with can-i
  • How the OAuth server + an identity provider authenticate users, and how htpasswd works
  • How LDAP/AD and OIDC integrate, and how group sync maps the directory onto RBAC
