Workloads, Storage & Networking — visualized

Five hands-on visualizations that make Module 3 click — how to pick the right workload controller, how a rolling update ships zero-downtime releases (and rolls back), how Services and DNS make pods discoverable, how ConfigMaps, Secrets and PVCs configure and persist your apps, and the special controllers — StatefulSets, DaemonSets and Jobs. Each ends with a short self-check quiz.

How to use this

Work through the five in order (each builds on the last) or jump to any concept. Everything runs in your browser — no install, no internet, no cluster needed. Companion reading: guides/module-03-workloads-storage-networking.md. Then practice for real with the guided demos and exercises using kubectl on a minikube cluster.

What you'll be able to explain afterwards

  • Which workload controller fits which job — Deployment, StatefulSet, DaemonSet, Job/CronJob — and the Deployment → ReplicaSet → Pod ownership chain
  • How rolling updates ship releases with no downtime, and how rollback returns to a prior revision in one command
  • How a Service + EndpointSlices + CoreDNS give pods a stable identity, and when to use ClusterIP vs NodePort vs Ingress
  • How ConfigMaps/Secrets decouple config from the image (and why a Secret is base64, not encrypted), and how a PVC persists data beyond a pod's life
  • What StatefulSets, DaemonSets and Jobs add that a Deployment can't
