Five hands-on, interactive visualizations that make Module 2's core concepts click — how a cluster is wired, how the control plane keeps your apps alive, how the scheduler places pods, what really happens when you create one, and how Services find the right pods. How to use this

Work through the five in order (each builds on the last) or jump to any concept. Everything runs in your browser — no install, no internet, no cluster needed. Companion reading: guides/module-02-kubernetes-architecture.md. Then practice for real with the guided demos and exercises using kubectl on a minikube cluster.

What you'll be able to explain afterwards

  • How a cluster splits into a control plane (brain) and worker nodes (muscle), and why every component talks through the API server
  • The roles of kube-apiserver, etcd, scheduler, controller-manager, kubelet, container runtime (CRI-O), and kube-proxy
  • The desired-state + reconciliation model — the source of Kubernetes self-healing and scaling
  • How the scheduler places pods using filtering (predicates) and scoring (priorities)
  • The end-to-end pod-creation flow, and how labels, Services, endpoints, and DNS deliver stable service discovery

Then take it to a real cluster

The visualizations build the mental model; these apply it on a real Kubernetes cluster (minikube) with kubectl:

  • Guided demos — instructor-led tours of the live cluster (explore architecture, watch self-healing, influence the scheduler, trace a pod's birth, service discovery).
  • Exercises — you drive, with validation steps and worked solutions.
  • Cheat sheets — quick kubectl command reference for inspecting cluster architecture.
