Concept · Operators via OLM

Install software that operates itself.

For complex, stateful software you don't hand-roll manifests — you install an Operator from OperatorHub. The Operator Lifecycle Manager turns your Subscription into a running Operator, and you deploy by creating a Custom Resource.

OLM is the package manager for Operators — it handles discovery, install, dependency resolution and upgrades. OperatorHub is its catalog front-end.

Then use it: create a Custom Resource

The Operator added CRDs to the API. You declare what you want; it builds & operates it.

kind: PostgresCluster
metadata: { name: cdr-store }
spec: { instances: 3, backups: { schedule: "0 2 * * *" } }
→ The Operator provisions a StatefulSet + PVCs + Services and runs nightly backups & failover forever. You wrote ~5 lines, not a database's worth of YAML.
