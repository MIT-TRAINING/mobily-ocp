Concept · GitOps

Git is the source of truth. Argo CD makes it so.

In GitOps, the cluster's desired state lives in Git, and Argo CD continuously reconciles the cluster to match. Commit a change and watch it sync — then drift the cluster directly and watch self-heal snap it back.

Argo CD compares Git (desired) with the cluster (live). Match → Synced. Differ → OutOfSync, then it applies Git. With selfHeal, even changes made directly on the cluster are reverted to Git.

The Argo CD Application

kind: Application
spec:
  source: { repoURL: …/self-care.git, path: overlays/prod, targetRevision: main }
  destination: { server: …default.svc, namespace: self-care-prod }
  syncPolicy: { automated: { prune: true, selfHeal: true } }
An Application maps a Git source (repo/path/revision) to a cluster/namespace destination. Rollback = git revert or point targetRevision at an older commit — Argo syncs the cluster back.
