GitOps, CI/CD & Service Mesh — visualized

Three hands-on visualizations that make Module 10 click — how GitOps / Argo CD keeps the cluster in sync with Git (and heals drift), how a CI/CD pipeline builds an image and hands off to GitOps to deploy, and how a Service Mesh uses sidecars to secure and canary-route traffic. Each ends with a short self-check quiz.

How to use this

Work through the three in order (each builds on the last) or jump to any concept. Everything runs in your browser — no install, no internet, no cluster needed. Companion reading: guides/module-10-gitops-cicd-servicemesh.md. Then practice for real with the guided demos and exercises (Argo CD / Pipelines / Service Mesh are installed via Operators — admin-led).

What you'll be able to explain afterwards

  • The four GitOps principles and how Argo CD reconciles the cluster to Git (sync, drift, self-heal)
  • The difference between CI and CD, and why GitOps is the CD half
  • The Jenkins → GitOps flow: build an image, update Git, let Argo CD deploy — with no cluster creds in CI
  • How a Service Mesh works via sidecars (data plane) + control plane, and what it adds without code changes
  • How traffic management (VirtualService weights) enables canary releases, observed in Kiali
